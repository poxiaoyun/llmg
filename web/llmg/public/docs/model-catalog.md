# 模型与价格

和公共模型目录不同，LLMG 中“你能看到什么模型、能不能调用、价格是多少”都依赖当前实例的实际配置。用户文档最重要的不是背一份固定模型表，而是知道去哪里查看当前实例的真实结果。

## 先看哪里

### 价格页

站内的 `/pricing` 页面适合做人工查看。你可以直接看到：

- 模型名称
- 上下文长度
- 输入 / 输出 / 缓存价格
- 能力标签

### `GET /v1/models`

如果你要在代码里动态发现模型，用 `/v1/models` 更合适：

```bash
curl https://your-llmg-domain.com/v1/models \
  -H "Authorization: Bearer YOUR_LLMG_API_KEY"
```

## 为什么不同实例看到的模型不一样

下面这些因素都会影响最终可见模型：

- 管理员启用了哪些渠道
- 渠道中实际暴露了哪些模型
- 是否做了模型映射或重命名
- 你的 key 是否受分组、权限或模型白名单限制

## 怎么选择第一个测试模型

如果你只是验证接入链路，优先选择：

- 成本较低的聊天模型
- 在价格页里明确可见的模型
- 不依赖特殊协议或多模态输入的模型

## 如何理解价格页里的字段

| 字段 | 作用 |
| --- | --- |
| Context | 大致可用上下文窗口 |
| Input price | 输入 token 计费 |
| Output price | 输出 token 计费 |
| Cache price | 缓存命中相关计费 |
| Capabilities | 文本、图像、音频、函数调用等能力标签 |

## 一个推荐流程

1. 在 `/pricing` 里挑一个明确可见的模型。
2. 用同样的模型 ID 发一次 `chat/completions` 或 `messages` 测试请求。
3. 如果返回模型不可用，再回价格页或 `/v1/models` 对照。

## 如果你需要程序化区分不同协议

- OpenAI 兼容调用：优先看 `/v1/models`
- Anthropic 兼容调用：仍可用 `/v1/models`，但带上 `x-api-key` 与 `anthropic-version`
- Gemini 风格客户端：可以使用 `/v1beta/models` 风格路径列出模型

## 常见误区

### 直接照抄别人文档里的模型名

别的实例可用，不代表你的实例也启用了同样的模型。

### 只看价格，不看能力标签

便宜不等于合适。上下文、模态支持和工具调用能力往往同样重要。

### 只看 `/v1/models`，不看价格页

程序化列表适合发现模型，价格页更适合人工判断成本和能力边界。
