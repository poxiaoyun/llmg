# 价格与计费

LLMG 的价格与计费界面主要服务两个目的：第一，向调用方清楚展示模型价格与能力；第二，把余额、充值、配额和调用消耗串到同一条成本链路里。

## 价格页可以确认什么

- 模型名称、上下文长度与能力标签。
- 输入、输出和缓存价格。
- 按量计费还是按次计费。
- 当前可用的供应商与端点类型。

## 控制台里的成本面板关注什么

在控制台概览页，优先关注三组信息：

- **余额**：当前账户还能继续支撑多久。
- **充值入口**：需要补充额度时的操作入口。
- **用量概览**：请求数、消耗金额、Token 规模是否符合预期。

## 计费策略的常见组合

### 按 Token 计费

适合大多数聊天、嵌入和生成模型。你会在价格页看到输入、输出以及缓存价格的拆分。

### 按请求计费

适合部分特殊模型或第三方上游的按次结算方式。通常要在策略上额外标记，避免前端误判为纯 Token 计费。

### 缓存计费

项目已经支持多类模型的缓存价格展示与结算。对缓存命中率较高的业务，这一栏能直接影响你对上游选型的判断。

## 运营建议

- 不要只看单模型单价，也要看上下文长度、缓存价格和真实调用模式。
- 把高频模型的价格变化纳入日常巡检，避免上游变价后前台展示滞后。
- 如果你开启了在线充值、订阅或支付回调，先跑通测试链路再暴露给真实用户。

## 何时回到价格页复核

- 新增供应商或更换渠道时。
- 调整模型映射、分组或计费倍率后。
- 发现某个应用成本突然升高时。
- 计划给外部客户或团队开放新的模型目录时。
